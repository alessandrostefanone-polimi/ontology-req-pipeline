[SCHEMA DEFINITION]

You MUST output a single JSON object with the following structure:

{
  "original_text": string,
  "structure": {
    "system_agent": string | null,
    "modal_verb": string | null,
    "precondition": string | null,
    "system_response": string | null
  },
  "constraints": [
    {
      "text_span": string,
      "target_attribute": string,
      "normalization": {
        "operator": string | null,
        "primary_value": number | null,
        "secondary_value": number | null,
        "tolerance": number | null,
        "unit": string | null
      }
    },
    ...
  ]
}

Field semantics:

- original_text (string):
  The full, original requirement text you were given (exactly as input, unchanged).

- structure (object):
  - system_agent (string | null):
    The part of the requirement that describes the system, subsystem, or entity to which the requirement applies. Do not include any articles like "the" or "a" in this field.
    If you cannot clearly identify it, set this field to null.
  - modal_verb (string | null):
    The modal verb used in the requirement (e.g., "must", "shall", "should").
    If there is no modal verb, set this field to null.
  - precondition (string | null):
    Any explicit condition, circumstance, or “when/if/during” clause under which the requirement applies.
    If there is no explicit precondition, set this field to null.
  - system_response (string | null):
    The behavioral or state requirement: what the system must/shall/should do or satisfy. Do not include any modal verbs like "must", "shall", or "should" here.
    If you cannot clearly identify it, set this field to null.

- constraints (list of objects):
  Each element represents a constraint (usually a numeric or quantitative condition) extracted from the requirement.

  - text_span (string):
    The exact substring from the original requirement that expresses the constraint (as close as possible).

  - target_attribute (string):
    The attribute of the system that this constraint refers to (e.g. "Section Mass (Scope Definition)", "Fastener Count", "Beam Mass", "Coolant Temperature").
    Use a concise, descriptive phrase; reuse the example names when appropriate.

  - normalization (object):
    A normalized view of the constraint’s quantitative information.

    - operator (string | null):
      A symbolic representation of the operator. Examples:
        - "Range"
        - "<="
        - "<"
        - ">="
        - ">"
        - "=="
      Choose the best-fitting operator. If you are unsure, use a best-effort symbolic approximation. If there truly is no operator, use null.

    - primary_value (number | null):
      The main numeric value. For a range, this is the lower bound; for a minimum, this is the minimum value; for an equality, this is the target value, etc.

    - secondary_value (number | null):
      The secondary numeric value when relevant (e.g., upper bound for a range); otherwise null.

    - tolerance (number | null):
      A symmetric tolerance around the primary_value when explicitly given (e.g., ±5). Otherwise null.

    - unit (string | null):
      The surface form of the unit (e.g., "kg", "N", "°C", "degC"). If there is no unit, set null.


[GENERAL GUIDELINES]

1. Preserve original_text exactly.
2. Do not invent constraints: only annotate what is explicitly present or very strongly implied by the wording.
3. When there are multiple quantitative conditions, create multiple constraint objects.
4. When there is no clear precondition, set "precondition": null.
5. It is acceptable to assign a generic but meaningful target_attribute (e.g., "Beam Mass", "Coolant Temperature", "Bolt Torque") when the requirement is clear but does not provide a canonical name.
6. Numeric values can be integers or decimals. Use a dot for decimals (e.g., 3.5).
7. If you cannot determine a value, or it does not exist, use null.

[SPECIAL CASES]

1. Angle Measurements:
   - Use "DEG" for degrees and "rad" for radians in the unit field.
2. Percentages:
   - Use "PERCENT" as the unit.
3. Dimensionless Quantities (e.g., ratios, coefficients):
   - Use "DimensionlessUnit" as the unit.
